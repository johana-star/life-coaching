Voice & tone

Core principles

Direct, warm, honest — not performative. Front-loads the most important information. Names systems plainly and takes positions. Discloses personal experience naturally, without over-explaining.

Naming convention

Name things after what they evoke, not what they describe. Applies to palette token names, CSS class names, section names, and copy. Examples: --mustard not --dark-yellow; --abyss not --darkest-blue.

Language preferences

Singular they

Johana uses they/them pronouns. The verb form depends on the subject:

SubjectVerbExample
theypluralThey are late-diagnosed. They have navigated the SDP.
Proper namesingularJohana is shadowing as an Independent Facilitator.

This mirrors how you works — "you are" whether one person or many, but "Alex is" when using the name. With practice, "they are" becomes natural. (Merriam-Webster)

Hue wheel

All 28 active tokens arranged by hue angle

Angle = hue (clockwise from top = 0°). Distance from center = darkness — lighter colors cluster near center, darker toward the edge. Circle size is proportional to the lightness step within each family. Small faded circles at 0°, 90°, 180°, and 270° show the pure hue at that angle for orientation — they are not palette tokens.

Body — DM Sans 300

p · 0.95–1.05rem · leading 1.75–1.85

Body copy is set in DM Sans at 300 weight. It is warm and direct — written to be read by someone who may be tired, overwhelmed, or skeptical. Line height is generous. Sentences are honest.

Banner & ribbon

The banner is an aria-hidden decorative stripe bar directly below the nav. Eight palette colors in a fixed sequence — sakura → cherry → cayenne → mustard → sage → coastal → abyss → lavender — functioning as a visual table of contents for the page's color journey. The ribbon is an identical stripe bar at 6px tall placed directly above the footer, reprising the same sequence as a closing callback. Both use the same .stripe and -bg classes; height is the only difference.

Banner — 3.75rem

Ribbon — 6px (0.375rem)

Color order mirrors the page's section progression — the banner says "here's what's coming," the ribbon says "here's what you saw." The sequence is fixed; do not reorder. Colors reference CSS custom properties so they participate in the token system despite being in decorative markup.

Spacing & layout

Section padding

Two tiers. Standard sections use 5rem 4rem (vertical × horizontal). Hero sections — those that anchor a major content shift — use 7rem 4rem. The horizontal 4rem is consistent across all sections and collapses to 2rem on mobile.

TierPaddingSections
Standard5rem 4rem#systems, #process
Hero7rem 4rem#about, #offerings, #testimonials, #diagnosis
Hero (exception)4rem#hero — full-viewport, padding uniform on all sides
Mobile (all)5rem 2remAll sections collapse to this below 900px

Inner max-width pattern

Full-bleed sections that contain a single column of content use an .inner wrapper with max-width and margin: 0 auto to constrain line length. Two-column grid sections (#about, #not-therapy, #hero) let the grid handle width — no .inner needed.

ContextMax-width
Body copy within a section (#systems p, #diagnosis p)640px
Section inner wrapper (.inner)860px
#testimonials inner1000px
#book layout1100px

Grid gaps

Column gaps are generous — content that breathes. Offering and testimonial card grids use 1px gap to create hairline dividers via the section background showing through.

ContextGap
#hero, #not-therapy column gap4rem
#about column gap6rem
#book column gap6rem
#offerings card grid1px (hairline divider)
#testimonials card grid1.5rem
#process step list0.25rem 2rem (row × column)

Breakpoint

One breakpoint at 900px. Below it: all multi-column grids collapse to single column, horizontal padding halves to 2rem, and the base font size steps up from 18px to 20px.
